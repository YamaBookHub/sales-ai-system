# 17_IMPLEMENTATION_ROADMAP

## 目的

営業獲得業務を、候補発見から受注後対応まで段階的にシステム化する。

## Phase 1: 営業リストとメール下書き

- CAMPFIRE URLを手入力して案件を取り込む。
- Company / Project / Lead を作成する。
- リード一覧を画面で確認する。
- OpenAI APIで営業メール下書きを作成する。
- 生成メールを画面で確認・編集する。
- request-review / approve / queue の承認フローを使う。
- 未承認メールは送信キューに入れない。
- 送信前チェックリストをメールごとに保存し、未完了なら承認・キュー投入を止める。

## Phase 2: 分析メモとチェックリスト

- 商品特徴、弱点、改善余地、SNSでの見せ方をリードに紐づけて記録する。
- 過去プロジェクト分析を記録する。
- ブランドHP / SNS分析を記録する。
- NG表現、実績表現、宛名確認などのチェック項目を拡張する。
- NG表現と実績表現チェックを行う。

## Phase 3: 候補探索と連絡先管理

- CAMPFIREの一覧ページから開催中プロジェクト候補を収集する。
- Makuake / GREEN FUNDING の取り込みを追加する。
- 会社HP、問い合わせフォーム、メールアドレス、サイト内連絡手段を記録する。
- A/B/C優先度とスコアリング基準を整備する。

## Phase 4: 参考メールとRAG

- 過去メールを成功例、失敗例、参考例として保存する。
- 件名、導入文、切り口、CTAのパターンを管理する。
- 営業感が弱い表現、怪しく見える表現、NG表現を管理する。
- メール生成時に参考メールを利用する。

## Phase 5: 送信後管理

- 送信日、送信手段、件名、切り口、CTA、次回確認日を記録する。
- Gmail連携または手動登録で返信を保存する。
- 返信を商談希望、興味あり、資料希望、断り、時期違いなどに分類する。
- 日程調整、質問回答、お礼メールのテンプレートを作る。

## Phase 6: 商談以降

- 面談前メモ、ヒアリング項目、提案方針を作る。
- 面談議事録、お礼メール、次回対応を保存する。
- 提案書、見積、契約、請求、キックオフ準備を管理する。

## 現在の優先順位

1. Phase 1を画面で使える状態にする。
2. Phase 2の分析メモと送信前チェックをDB化する。
3. Phase 3の連絡先管理を追加する。
4. その後に候補探索、RAG、送信後管理へ進む。
