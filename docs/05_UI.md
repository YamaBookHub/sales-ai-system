Version: 1.0  
Status: Draft  
Target: MVP / Version 1

---

## 1. 本書の目的

本書は、営業AIシステムにおける画面構成、画面遷移、各画面の表示項目、操作項目、権限、API連携方針を定義する。

本システムでは、AIが生成した分析結果・営業メールをそのまま使用するのではなく、人間が確認・修正・承認できるUIを前提とする。

---

## 2. UI設計原則

### 2.1 Human in the Loop

AIは分析・提案・下書きを行う。  
人間は確認・修正・承認を行う。

AIが生成した内容は、すべて画面上で編集できるものとする。

### 2.2 AI信頼度の表示

すべてのAI分析結果には、AI信頼度を表示する。

| 信頼度 | 意味 |
|---|---|
| 高 | 商品情報・ブランド情報・過去プロジェクト情報が十分取得できている |
| 中 | 一部不足はあるが、営業メール作成は可能 |
| 低 | 情報不足が多く、追加調査が必要 |

### 2.3 根拠の表示

AIの分析結果には、可能な限り根拠を表示する。

例：

```text
根拠:
- 商品ページ内に「軽量」「防水」と記載あり
- 過去プロジェクト5件を取得済み
- 支援総額、支援者数、達成率を取得済み
```

### 2.4 編集可能性

以下はすべて編集可能とする。

- ブランド分析
- 商品分析
- 過去プロジェクト分析
- SNS分析
- 営業切り口
- 件名
- メール本文
- 品質チェック結果に対するメモ
- ステータス
- 優先度

### 2.5 誤送信防止

MVPではメール送信機能は実装しない。  
ただし、将来の送信機能を前提に、送信前レビュー画面を設計する。

## 7. UI-002 New Analysis

### 7.1 目的

クラウドファンディングのプロジェクトURLを入力し、分析を開始する。

### 7.2 入力項目

| 項目 | 必須 | 内容 |
|---|---|---|
| projectUrl | 必須 | CAMPFIRE等のプロジェクトURL |

### 7.3 操作

- 分析開始
- クリア

### 7.4 バリデーション

- URLが空ではない
- 対応サイトURLである
- プロジェクトIDを含む

### 7.5 エラー表示

| 条件 | メッセージ |
|---|---|
| URL未入力 | URLを入力してください |
| 非対応URL | 対応していないURLです |
| 取得失敗 | プロジェクト情報を取得できませんでした |
| AI分析失敗 | AI分析に失敗しました |

### 7.6 使用API

- `POST /api/run`

## 15. UI-010 Quality Check

- 「訴求」の使用

## 16. UI-011 Review

### 16.1 目的

送信前に人間が最終確認する。

### 16.2 確認項目

- 企業名が正しい
- 商品名が正しい
- 商品理解が正しい
- 営業切り口が適切
- 実績表現が正しい
- CTAが自然
- 品質チェックが完了している

### 16.3 操作

- 承認
- 差し戻し
- 編集へ戻る

MVPではメール送信は行わず、承認済み状態まで管理する。

### 16.4 使用API

- `POST /api/projects/:id/approve`
- `POST /api/projects/:id/reject`

## 20. 権限別UI

| 機能 | 営業担当 | 上長 | 管理者 |
|---|---|---|---|
| URL登録 | 可 | 可 | 可 |
| 分析確認 | 可 | 可 | 可 |
| 分析編集 | 可 | 可 | 可 |
| メール編集 | 可 | 可 | 可 |
| 承認 | 不可 | 可 | 可 |
| 設定変更 | 不可 | 不可 | 可 |
| ナレッジ管理 | 一部可 | 可 | 可 |

- 履歴を確認できる
