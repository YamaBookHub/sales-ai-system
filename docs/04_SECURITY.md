# 04_SECURITY.md

## セキュリティ・コンプライアンス仕様

## 1. 結論
個人情報、メール本文、外部API認証情報を扱うため、RBAC、監査ログ、配信停止、重複送信防止を必須とする。

## 2. RBAC
| 操作 | admin | manager | operator | viewer |
|---|---:|---:|---:|---:|
| 閲覧 | ✅ | ✅ | ✅ | ✅ |
| Lead編集 | ✅ | ✅ | ✅ | ❌ |
| AI生成 | ✅ | ✅ | ✅ | ❌ |
| Mail承認 | ✅ | ✅ | ❌ | ❌ |
| 送信キュー投入 | ✅ | ✅ | ❌ | ❌ |
| 設定変更 | ✅ | ❌ | ❌ | ❌ |

## 3. 送信安全要件
- `MAIL_SEND_ENABLED=false` なら実送信不可。
- 配信停止済みContactへ送信不可。
- Blocked Companyへ送信不可。
- AI生成メールは承認なしで送信不可。
- 送信・承認・配信停止はAuditLogまたはEmailEventに保存。

## 4. 個人情報
IPはHash保存を原則とする。開封・クリックは参考指標であり、過度な追跡判断に使わない。

## 5. AI安全要件
AIは実績、数値、制度、日付、社名を創作してはいけない。不明な情報は不明として扱う。

## Codex実装条件
- docs/ を正本として実装する。
- DB変更時は `docs/07_DATABASE.md` と `prisma/schema.prisma` を同時更新する。
- API変更時は `docs/06_API.md` と `openapi/openapi.yaml` を同時更新する。
- AI生成・メール送信・承認・配信停止は必ずログを残す。
- MVPではAI生成メールの自動送信は禁止。人間承認を必須にする。
