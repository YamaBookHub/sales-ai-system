# 00_CHARTER.md

## 営業AIシステム v1.0 プロジェクト憲章

## 1. 結論
本システムは、クラウドファンディング掲載企業を中心に、営業候補の登録、優先度判定、AIメール下書き、承認、送信、返信分類、次アクション管理までを一気通貫で扱う社内営業支援システムである。

## 2. 目的
- 営業候補探しを属人作業から管理可能な業務にする。
- メール品質をテンプレートとAIで安定させる。
- 送信・返信・クリック履歴を残し、対応漏れを防ぐ。
- Codexが実装開始できる仕様群を整備する。

## 3. MVPスコープ
| 対象 | 内容 |
|---|---|
| Lead管理 | 企業、案件、営業状態、優先度 |
| AI | メール下書き、返信分類、スコア補助 |
| Mail | 下書き、承認、キュー、Gmail送信 |
| Tracking | 開封、クリック、配信停止 |
| Task | 返信後の次アクション |

## 4. 対象外
完全自動スクレイピング、完全自動送信、契約・請求、商談議事録、SaaS化はMVP対象外。

## 5. 技術方針
Next.js、NestJS、Prisma、PostgreSQL、BullMQ、Redis、Gmail API、AI Provider Adapterで構成する。

## 6. 成功条件
- リード登録から送信・返信確認まで操作できる。
- 承認なし送信ができない。
- DB/API/AI/Mail仕様が矛盾しない。
- GitHubにそのまま反映できる成果物になっている。


---

## Codex実装条件
- docs/ を正本として実装する。
- DB変更時は `docs/07_DATABASE.md` と `prisma/schema.prisma` を同時更新する。
- API変更時は `docs/06_API.md` と `openapi/openapi.yaml` を同時更新する。
- AI生成・メール送信・承認・配信停止は必ずログを残す。
- MVPではAI生成メールの自動送信は禁止。人間承認を必須にする。
